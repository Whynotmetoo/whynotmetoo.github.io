Introducing atmos-fx

Published June 12, 2026

  • #announcement
  • #ui
  • #design
  • #frontend

When I was looking for a new project idea, I realized that I have always liked interfaces with atmosphere.

Not just clean layouts or polished components, but small environmental details that make a page feel alive: rain moving behind content, particles reacting to elements, drops falling and splashing against a card, or a subtle weather layer that changes the mood of a page.

Apple Weather was one of the inspirations behind this idea. I like how weather is not only displayed as data, but also presented as a visual environment. Rain, snow, fog, light, and motion all become part of the interface. It is satisfying just watching it move.

So I started building atmos-fx.

What Is atmos-fx?

atmos-fx is a small WebGL-based atmosphere engine for the web.

It lets you add weather-inspired visual effects to DOM interfaces, with particles that can interact with page elements. The current focus is rain: drops falling through the scene, colliding with selected elements, and creating small splash effects.

The library has no dependencies. It is built from scratch, mainly as an experiment in making UI feel more atmospheric and playful.

It might not be a practical UI component library in the traditional sense. It is more for creative development, experimental interfaces, personal websites, weather-themed pages, landing pages, and visual component showcases.

How to Use

Install it from npm:

npm i atmos-fx

If you are using React, you can wrap part of your UI with AtmosFx and use AtmosCard for elements that should visually interact with the atmosphere layer:

import { AtmosFx, AtmosCard } from 'atmos-fx'
import 'atmos-fx/styles.css'

export function App() {
  return (
    <AtmosFx preset="rain" density={0.65} wind={-0.12} bottomCollision>
      <div className="layout">
        <AtmosCard transMode="glass" liquidDripping>
          <h1>Interactive Weather UI</h1>
          <p>Precipitation can splash and react to this card.</p>
        </AtmosCard>
      </div>
    </AtmosFx>
  )
}

You can also use asChild when you want to avoid rendering an extra wrapper element.

You can also use it directly with Vanilla JavaScript:

<div id="container">
  <div data-atmos-collision data-atmos-glass data-atmos-liquid-dripping="true">
    <h1>Interactive Weather UI</h1>
    <p>Precipitation can splash and react to this element.</p>
  </div>
</div>
import { createAtmosphere } from 'atmos-fx'
import 'atmos-fx/styles.css'

const container = document.querySelector('#container')

const controller = createAtmosphere(container, {
  preset: 'rain',
  density: 0.65,
  wind: -0.12,
  bottomCollision: true,
})

controller.start()

What It Can Be Used For

I think atmos-fx is especially suited for:

  • creative component showcases
  • weather-themed landing pages
  • personal websites with atmospheric effects
  • playful UI experiments
  • interactive backgrounds
  • visual demos that do not need to be purely practical

It is probably not something every production dashboard needs.

But for projects where mood, motion, and visual identity matter, I think atmos-fx can make a page feel more memorable.

What Comes Next

The current version was just released, and this is only the beginning.

I may add more weather options and animation styles later, such as fog, wind, hail, even sunny day, or other atmosphere-driven effects. I also want to keep exploring how visual effects can interact with real DOM elements without forcing developers to rebuild their UI around a special rendering system.

You can check it out here:

https://atmosfx.carsonye.com/
