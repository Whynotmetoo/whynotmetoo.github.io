Static publishing pipeline from private source to public Pages

Published February 10, 2026

Updated February 12, 2026

  • #astro
  • #github-actions
  • #deployment

This post demonstrates the deployment model used by this site:

  1. Author content in a private source repository.
  2. Build static files in CI.
  3. Push only dist/ artifacts into a public Pages repository.

Why this setup works

  • Source code can stay private while published output remains public.
  • No runtime server and no database are required.
  • Rollback is simple because the public repo is just static files.

Workflow excerpt

- name: Build
  run: npm run build

- name: Checkout public Pages repo
  uses: actions/checkout@v4
  with:
    repository: whynotmetoo/whynotmetoo.github.io
    token: ${{ secrets.PAGES_REPO_TOKEN }}

Keep token scope minimal. Contents: Read and write on the target Pages repo is enough.

Structure notes

astro_blog is the authoring repository and whynotmetoo.github.io is the publish repository.

  • Authoring concerns: content, layout, components, CI definitions.
  • Publish concerns: built HTML, CSS, assets, and .nojekyll.
